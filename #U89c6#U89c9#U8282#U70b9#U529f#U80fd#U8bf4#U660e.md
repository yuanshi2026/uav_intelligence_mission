# 视觉节点功能说明

本文档说明 `scripts/stage1_vision_node.py` 的功能、接口、算法流程和调试参数。

## 1. 节点定位

`stage1_vision_node.py` 是第一阶段任务的统一视觉节点，负责响应 FSM 的视觉请求，但不直接控制无人机，也不直接投放。

它只做：

```text
相机取图 -> 识别目标 -> 计算机体系米制偏差 -> 发布给 FSM
```

FSM 仍然负责：

```text
状态切换 -> 航点移动 -> 对准控制 -> 投放命令 -> 穿环航点生成
```

## 2. 相机分工

| 相机 | 用途 |
|---|---|
| 下视 USB 相机 | 扫描二维码、判断图片靶类别、定位特殊靶中心 |
| 前视 Intel RealSense D435i | 只在穿环阶段定位圆环 |

对应 `scan_target`：

| `scan_target` | 使用相机 | 功能 |
|---|---|---|
| `qr` | 下视相机 | 混合二维码扫码 |
| `image_target` | 下视相机 | 方形靶板定位 + YOLO 图片类别分类 |
| `special_target` | 下视相机 | 方形特殊靶定位 |
| `ring_gate` | 前视 RealSense | 圆环中心定位 |
| `none` | 无 | 不识别 |

## 3. 题目视觉要求摘要

根据比赛规则：

- 二维码大小为 `200mm x 200mm`，内容为两个英文类别和 `left/right`，例如 `man,apple,left`。
- 图片靶大小为 `800mm x 800mm`，外圈圆环直径 `600mm`，内圈圆环直径 `400mm`，中间图片来自 CIFAR-100。
- 特殊靶大小为 `800mm x 800mm`，同样有环形区域，四角有不同步闪烁的信号灯。
- 圆环外径 `1.2m`，内径 `0.9m`，底部距地面 `1m`，位置不固定。
- 飞行高度一般不得低于 `1.2m`，悬停投放时无人机高度不得高于 `0.8m`。

节点据此默认使用：

- 图片靶和特殊靶：`0.80m` 方形靶板做透视定位。
- 圆环：`1.20m` 外径做前视估距。

## 4. ROS 接口

### 4.1 订阅

| 话题 | 类型 | 说明 |
|---|---|---|
| `/uav/scan_enable` | `std_msgs/Bool` | FSM 是否需要视觉工作 |
| `/uav/scan_target` | `std_msgs/String` | 当前识别目标 |

只有当：

```text
/uav/scan_enable = true
```

并且 `scan_target` 不是 `none` 时，节点才会执行对应识别。

### 4.2 发布

| 话题 | 类型 | 说明 |
|---|---|---|
| `/uav/qr_text` | `std_msgs/String` | 二维码文本，格式 `class1,class2,left/right` |
| `/uav/vision_result` | `std_msgs/String` | 图片靶、特殊靶、圆环统一 JSON 结果 |

## 5. 二维码识别

二维码由下视相机识别。

处理流程：

1. 读取下视相机画面。
2. 优先使用 OpenCV `QRCodeDetector`。
3. 对图像做灰度、CLAHE、锐化、自适应阈值等轻量增强后重复扫码。
4. 如果安装了 `pyzbar`，再用 `pyzbar` 兜底扫码。
5. 将结果规范化为小写、无空格的：

```text
class1,class2,left
```

或：

```text
class1,class2,right
```

发布示例：

```text
/uav/qr_text = "man,apple,left"
```

二维码阶段不发布 `/uav/vision_result`，因为 FSM 目前读取的是旧接口 `/uav/qr_text`。

## 6. 图片靶识别

图片靶使用下视相机。

流程：

1. 用 OpenCV 找画面里的方形靶板轮廓。
2. 将四边形角点排序为 `TL/TR/BR/BL`。
3. 通过已知靶板大小 `0.80m x 0.80m` 建立透视变换。
4. 将投放中心像素点映射到靶板平面坐标。
5. 根据靶板中心和投放中心的差，计算：

```text
offset_x_m：目标中心相对无人机前后方向偏差
offset_y_m：目标中心相对无人机左右方向偏差
```

6. 将靶板透视矫正为正方形图片。
7. 默认裁剪靶板中心区域，送入 YOLO 分类模型。
8. 发布 `image_target` JSON。

发布示例：

```json
{
  "target": "image_target",
  "detected": true,
  "class_name": "apple",
  "offset_x_m": 0.04,
  "offset_y_m": -0.03,
  "confidence": 0.86,
  "stable_count": 5,
  "target_center_px": [318.0, 241.0],
  "release_center_px": [320.0, 240.0],
  "offset_px": [-2.0, 1.0],
  "mapping_method": "square_homography_800mm",
  "classifier_method": "yolo_classify"
}
```

## 7. YOLO 模型占位

YOLO 模型占位写在节点文件最前面：

```python
IMAGE_CLASS_YOLO_MODEL_PATH = "/home/nvidia/catkin_ws/src/uav_inventory/models/image_target_classifier.engine"
IMAGE_CLASS_YOLO_TASK = "classify"
IMAGE_CLASS_YOLO_CONF = 0.25
```

运行时也可以通过 launch 参数覆盖：

```xml
<arg name="image_class_model_path"
     default="/home/nvidia/catkin_ws/src/uav_inventory/models/image_target_classifier.engine" />
<arg name="image_class_model_task" default="classify" />
```

当前版本只加载一个 YOLO 模型，用于图片靶类别分类。

当前默认路径使用 TensorRT engine 文件：

```text
image_target_classifier.engine
```

如果你的 engine 是由 YOLO 分类模型导出的，`image_class_model_task` 保持 `classify`。如果后续你换成检测模型导出的 engine，再把该参数改为 `detect`。

后续如果时间允许，可以增加：

- 靶板检测 YOLO 模型：替代 OpenCV 方形轮廓。
- 特殊靶检测 YOLO 模型：更稳定定位特殊靶。
- 圆环检测 YOLO 模型：替代前视圆形轮廓检测。

代码中 `classify_image_target()` 已经兼容一部分 detection 输出，但默认设计仍是分类模型。

## 8. 特殊靶定位

特殊靶也使用下视相机。

当前版本没有使用 YOLO 找特殊靶，而是利用题目规则：

```text
特殊靶是 800mm x 800mm 方形靶板
```

处理流程：

1. OpenCV 找最大、较正、接近方形的靶板轮廓。
2. 用 `0.80m` 方形靶板建立透视定位。
3. 靶板中心就是特殊靶中心。
4. 输出 `offset_x_m/offset_y_m`。

发布示例：

```json
{
  "target": "special_target",
  "detected": true,
  "offset_x_m": 0.02,
  "offset_y_m": -0.01,
  "confidence": 0.78,
  "stable_count": 4,
  "detector_method": "opencv_square"
}
```

如果现场光照复杂或方形轮廓不稳，后续建议加一个专门的特殊靶检测模型。

## 9. 圆环定位

圆环只使用前视 RealSense D435i。

处理流程：

1. 只在 `scan_target=ring_gate` 时读取和处理前视画面。
2. 对前视彩色图做灰度、模糊、Canny 边缘、闭运算。
3. 搜索近似圆形或椭圆形的大轮廓。
4. 找不到时用 `HoughCircles` 兜底。
5. 得到圆环中心像素和半径。
6. 默认用圆环已知外径 `1.20m` 和像素直径估算前向距离：

```text
forward_m = ring_outer_diameter_m * fx / ring_diameter_px
```

7. 用 RealSense 内参把像素偏移换算为：

```text
offset_y_m：圆环中心相对无人机左/右偏差
offset_z_m：圆环中心相对无人机上/下偏差
```

发布示例：

```json
{
  "target": "ring_gate",
  "detected": true,
  "forward_m": 1.20,
  "offset_y_m": -0.05,
  "offset_z_m": 0.03,
  "confidence": 0.72,
  "stable_count": 4,
  "ring_center_px": [315.0, 236.0],
  "ring_radius_px": 128.0,
  "mapping_method": "known_outer_diameter"
}
```

参数 `ring_use_depth` 默认为 `false`。原因是圆环中心是空洞，深度相机在中心像素上可能看到背景，不一定是圆环平面。现场如果背景深度可靠，可以设为 `true` 试用。

## 10. 坐标正负号

FSM 要求所有控制字段都是当前无人机机体系下的米制偏差。

图片靶和特殊靶：

```text
offset_x_m > 0：目标在无人机机头前方
offset_x_m < 0：目标在无人机机头后方
offset_y_m > 0：目标在无人机左侧
offset_y_m < 0：目标在无人机右侧
```

圆环：

```text
forward_m > 0：圆环在无人机前方
offset_y_m > 0：圆环在无人机左侧
offset_z_m > 0：圆环在无人机上方
```

下视相机默认安装假设：

```text
图像上方 = 无人机前方
图像左方 = 无人机左方
```

对应参数：

```xml
<param name="body_forward_from_image_down" value="-1.0" />
<param name="body_left_from_image_right" value="-1.0" />
```

如果现场测试发现正负号反了，优先改这两个参数。

前视 RealSense 默认：

```text
RealSense 图像右方为 camera_right
FSM 左方为 -camera_right
RealSense 图像下方为 camera_down
FSM 上方为 -camera_down
```

对应参数：

```text
ring_left_from_camera_right = -1.0
ring_up_from_camera_down = -1.0
```

## 11. 重要参数

### 11.1 下视相机

| 参数 | 默认值 | 说明 |
|---|---:|---|
| `down_camera_index` | `0` | 下视 USB 相机编号 |
| `down_width` | `640` | 图像宽度 |
| `down_height` | `480` | 图像高度 |
| `down_fps` | `30` | 采集帧率 |
| `down_auto_exposure` | `3` | 沿用旧节点自动曝光参数 |
| `down_gain` | `56` | 沿用旧节点增益参数 |
| `down_release_u` | `-1` | 投放中心 u，-1 表示图像中心 |
| `down_release_v` | `-1` | 投放中心 v，-1 表示图像中心 |
| `down_board_size_m` | `0.80` | 图片靶/特殊靶方形尺寸 |

### 11.2 方形靶板检测

| 参数 | 默认值 | 说明 |
|---|---:|---|
| `min_square_area_px` | `5000` | 最小方形面积 |
| `square_aspect_min` | `0.65` | 宽高比下限 |
| `square_aspect_max` | `1.45` | 宽高比上限 |
| `square_center_weight` | `0.15` | 候选方形靠近画面中心的加权 |
| `image_class_crop_ratio` | `0.50` | 送给 YOLO 的中心裁剪比例 |

### 11.3 圆环

| 参数 | 默认值 | 说明 |
|---|---:|---|
| `enable_front_realsense` | `true` | 是否启用前视 RealSense |
| `front_width` | `848` | RealSense 彩色/深度宽度 |
| `front_height` | `480` | RealSense 彩色/深度高度 |
| `front_fps` | `30` | RealSense 帧率 |
| `ring_outer_diameter_m` | `1.20` | 圆环外径 |
| `ring_use_depth` | `false` | 是否优先使用中心深度 |
| `ring_min_radius_px` | `35` | 最小圆环半径像素 |
| `ring_max_radius_px` | `360` | 最大圆环半径像素 |
| `ring_min_circularity` | `0.30` | 圆形度下限 |

## 12. 启动方法

实飞一键启动：

```bash
roslaunch uav_inventory stage1_real_flight.launch
```

这个总 launch 会同时启动：

```text
mavros px4.launch
livox_ros_driver2 msg_MID360.launch
faster_lio rflysim.launch
micro_uav_stage1_fsm.py
stage1_vision_node.py
udp_uav_cmd_receiver.py
```

指定下视相机和 engine 模型路径：

```bash
roslaunch uav_inventory stage1_real_flight.launch \
  down_camera_index:=0 \
  image_class_model_path:=/home/nvidia/catkin_ws/src/uav_inventory/models/best.engine
```

打开调试窗口：

```bash
roslaunch uav_inventory stage1_real_flight.launch show_debug:=true
```

只单独调视觉时，可以关闭外部飞控和定位节点：

```bash
roslaunch uav_inventory stage1_real_flight.launch \
  start_mavros:=false \
  start_livox:=false \
  start_faster_lio:=false \
  start_fsm:=false \
  start_udp_receiver:=false \
  start_vision:=true
```

## 13. 单独测试

### 13.1 测二维码

```bash
rostopic pub /uav/scan_enable std_msgs/Bool "data: true"
rostopic pub /uav/scan_target std_msgs/String "data: 'qr'"
rostopic echo /uav/qr_text
```

期望看到：

```text
man,apple,left
```

### 13.2 测图片靶

```bash
rostopic pub /uav/scan_enable std_msgs/Bool "data: true"
rostopic pub /uav/scan_target std_msgs/String "data: 'image_target'"
rostopic echo /uav/vision_result
```

检查：

- `detected=true`
- `class_name` 是否和图片类别一致
- `offset_x_m/offset_y_m` 正负号是否符合 FSM 约定
- `stable_count` 是否能稳定增长

### 13.3 测特殊靶

```bash
rostopic pub /uav/scan_target std_msgs/String "data: 'special_target'"
rostopic echo /uav/vision_result
```

检查特殊靶中心在画面中心时：

```text
offset_x_m ≈ 0
offset_y_m ≈ 0
```

### 13.4 测圆环

```bash
rostopic pub /uav/scan_target std_msgs/String "data: 'ring_gate'"
rostopic echo /uav/vision_result
```

检查：

- 圆环正前方时 `offset_y_m ≈ 0`
- 圆环偏左时 `offset_y_m > 0`
- 圆环偏右时 `offset_y_m < 0`
- 圆环偏上时 `offset_z_m > 0`
- 圆环偏下时 `offset_z_m < 0`

## 14. 现场调试建议

1. 先只开下视相机，确认 `qr` 能稳定发布 `/uav/qr_text`。
2. 再测 `special_target`，因为它不依赖 YOLO，只验证方形轮廓和米制偏差。
3. 手动移动靶板，确认前后左右正负号。
4. 接入 YOLO 分类模型，测 `image_target` 的类别名是否和二维码类别命名一致。
5. 最后测前视圆环，先看 `ring_center_px/ring_radius_px` 是否稳定，再看米制偏差。
6. 如果图片靶或特殊靶找不到方形，优先调 `min_square_area_px` 和现场光照。
7. 如果投放偏差固定偏一边，优先调 `down_release_u/down_release_v`。
8. 如果圆环距离不准，优先调 `ring_outer_diameter_m`、相机分辨率和圆环外轮廓检测。

## 15. 当前版本限制

- 图片靶类别依赖一个 YOLO 分类模型；模型文件需要你后续替换为真实路径。
- 图片靶和特殊靶定位依赖 OpenCV 方形轮廓，强反光或背景杂乱时可能需要专门的检测模型。
- 圆环定位目前是传统视觉轮廓法，复杂背景下可能不如专门训练的圆环检测模型。
- RealSense 深度默认不用于圆环中心距离，因为圆环中心是空洞，中心深度可能是背景。

## 16. 后续扩展点

建议后续优先做三项：

1. 增加靶板检测 YOLO 模型：输出图片靶/特殊靶方框或角点。
2. 增加特殊靶检测模型：直接定位特殊靶中心或环形中心。
3. 增加圆环检测模型：前视输出圆环外框、中心和置信度，再结合已知外径或深度估距。

节点前面的 YOLO 占位区已经为这些扩展留了位置。
